Authenticationâ€‹

Ring Platform uses NextAuth.js v5 with multiple authentication providers:

  • Google OAuth - Social login
  • Apple ID - Social login
  • MetaMask/Web3 - Blockchain wallet authentication
  • Credentials - Email/password authentication

Response Formatâ€‹

All API responses follow a consistent format:

Success Responseâ€‹

{
  "success": true,
  "data": { /* response data */ },
  "message": "Operation completed successfully"
}

Error Responseâ€‹

{
  "error": "ERROR_CODE",
  "message": "Human-readable error message",
  "code": 400,
  "details": { /* additional error details */ }
}

Rate Limitingâ€‹

  • Public endpoints: 60 requests/minute
  • Authenticated users: 120 requests/minute

ğŸš€ Quick Start Guideâ€‹

1. Authentication Setupâ€‹

// Install NextAuth.js
npm install next-auth

// Configure providers
import NextAuth from 'next-auth'
import GoogleProvider from 'next-auth/providers/google'

export default NextAuth({
  providers: [
    GoogleProvider({
      clientId: process.env.GOOGLE_CLIENT_ID,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET,
    })
  ]
})

2. Make Your First API Callâ€‹

// Get platform information
const response = await fetch('/api/info')
const platformInfo = await response.json()
console.log(platformInfo)

3. Create a User Profileâ€‹

// Update user profile
const profileData = {
  name: 'John Doe',
  bio: 'Software engineer',
  location: 'Kyiv, Ukraine'
}

const response = await fetch('/api/profile', {
  method: 'PUT',
  headers: {
    'Authorization': `Bearer ${sessionToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(profileData)
})
