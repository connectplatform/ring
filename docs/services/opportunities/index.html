Create Opportunity‚Äã

This document outlines the createOpportunity function, which allows authenticated users to create new opportunities in Firestore. The function enforces role-based access control and ensures data confidentiality.

Function Overview‚Äã

async function createOpportunity(data: Opportunity): Promise<Opportunity>

Creates a new opportunity in Firestore with validations for user role, confidentiality, and entity association.

Parameters‚Äã

Input‚Äã

NameTypeDescription
dataOpportunityThe opportunity data to be created.

Opportunity Object Structure‚Äã

Response‚Äã

Success‚Äã

Returns a Promise resolving to the newly created opportunity object:

interface Opportunity {
  id: string;
  type: OpportunityType;
  title: string;
  isConfidential: boolean;
  briefDescription: string;
  fullDescription?: string;

    min: number;
    max: number;

  };
  requiredSkills: string[];
  requiredDocuments: string[];
  attachments: Attachment[];
  visibility: OpportunityVisibility;
  contactInfo: {
    linkedEntity: string;
    contactAccount: string;
  };
}

Error‚Äã

Throws an error if any of the following occur:

  • Unauthorized access.
  • Invalid user role.
  • Entity does not exist.
  • Confidentiality validation fails.
  • Unauthorized association with an entity.

Implementation Details‚Äã

Steps in the Function‚Äã

  1. Authenticate User: Validates the session using auth().
  2. Role Validation:
    • Restricts VISITOR role from creating opportunities.
    • Ensures confidentiality rules are enforced.
  3. Entity Validation: Verifies the existence of the associated entity and its confidentiality rules.
  4. Authorization Check: Confirms the user creating the opportunity is linked to the associated entity.
  5. Firestore Write: Adds the new opportunity to the Firestore database.
  6. Return Response: Returns the newly created opportunity object with its autogenerated ID.

Get Opportunities‚Äã

This service fetches a paginated list of opportunities, respecting the assigned user role and visibility restrictions.

Function Overview‚Äã

getOpportunities(
  userRole: UserRole = UserRole.SUBSCRIBER,
  limit: number = 20,
  startAfter?: string
): Promise<{ opportunities: Opportunity[]; lastVisible: string | null }>

Request Parameters‚Äã

  • userRole: (Optional) UserRole Default: SUBSCRIBER The role of the user requesting opportunities. Supports roles like ADMIN, CONFIDENTIAL, MEMBER, etc.

  • limit: (Optional) number Default: 20 The maximum number of opportunities to retrieve per request.

  • startAfter: (Optional) string A Firestore document ID specifying where to start the pagination.

Response‚Äã

{
  opportunities: Opportunity[];
  lastVisible: string | null;
}

Response Details‚Äã

  • opportunities An array of Opportunity objects, including only those visible based on user roles (e.g., PUBLIC, SUBSCRIBER, or CONFIDENTIAL).

  • lastVisible A string containing the ID of the last visible document for pagination purposes. Null if no more documents exist.

Example Usage‚Äã

const { opportunities, lastVisible } = await getOpportunities(
  UserRole.MEMBER,
  10,
  'lastDocumentId'
);

console.log(opportunities);  // List of retrieved opportunities
console.log(lastVisible);    // ID of the last visible item

Get Confidential Opportunities‚Äã

This service fetches all opportunities marked as confidential. This function is designed specifically for users with roles like ADMIN or CONFIDENTIAL.

Authentication Requirement‚Äã

Only accessible by users assigned the ADMIN or CONFIDENTIAL roles.

Function Overview‚Äã

getConfidentialOpportunities(
  userRole: UserRole
): Promise<Opportunity[]>

Returns an array of Opportunity objects containing isConfidential set to true.

[

  • Includes Firestore document IDs (id) for each result.

Service Implementations‚Äã

Create Opportunity‚Äã

import { getOpportunitiesCollection, getEntitiesCollection } from '@/lib/firestore-collections';
import { Opportunity } from '@/features/opportunities/types';
import { auth } from '@/auth'; // Use the Auth.js v5 handler to get the session
import { UserRole } from '@/features/auth/types';

/**
 * Create a new Opportunity in Firestore, ensuring confidentiality and role-based rules.
 */
export async function createOpportunity(data: Opportunity): Promise<Opportunity> {
  try {
    console.log('Services: createOpportunity - Starting creation of new opportunity...');

    // Step 1: Authenticate and get user session
    const session = await auth(); // Auth.js v5 session handler
    if (!session || !session.user) {
      throw new Error('Unauthorized access');
    }

    const { id: userId, role: userRole } = session.user;

    // Step 2: Validate user role and permissions
    if (userRole === 'VISITOR') {
      throw new Error('Access denied. Visitors cannot create opportunities.');
    }

    if (data.isConfidential && userRole !== 'CONFIDENTIAL' && userRole !== 'ADMIN') {
      throw new Error('Only CONFIDENTIAL or ADMIN users can create confidential opportunities.');
    }

    console.log(`Services: createOpportunity - User authenticated with ID ${userId} and role ${userRole}`);

    // Step 3: Firestore setup
    const opportunitiesCollection = await getOpportunitiesCollection();
    const entitiesCollection = await getEntitiesCollection();

    // Step 4: Validate the associated Entity
    const entityId = data.contactInfo.linkedEntity; // Associated entity ID from contactInfo
    const entityDoc = await entitiesCollection.doc(entityId).get();

    if (!entityDoc.exists) {
      throw new Error(`Entity with ID ${entityId} not found.`);
    }

    const entity = entityDoc.data();
    const isEntityConfidential = entity?.isConfidential || false;
    const entityAddedBy = entity?.contactAccount; // Use the `contactAccount` field from the entity

    if (data.isConfidential) {
      // Check if the associated entity is also confidential and created by CONFIDENTIAL or ADMIN
      if (!isEntityConfidential || (entityAddedBy !== 'CONFIDENTIAL' && entityAddedBy !== 'ADMIN')) {
        throw new Error(
          'Confidential opportunities must be linked to confidential entities created by ADMIN or CONFIDENTIAL users.'
        );
      }
    }

    console.log(`Services: createOpportunity - Associated entity validated with ID ${entityId}`);

    // Step 5: Validate opportunity author relation with entity
    if (data.contactInfo.contactAccount !== userId) {
      throw new Error('You are not authorized to create an opportunity for this entity.');
    }

    // Step 6: Add the Opportunity to Firestore
    const newOpportunityData = {
      ...data,
      createdBy: userId,
      dateCreated: new Date(),
      dateUpdated: new Date(),
    };

    const docRef = await opportunitiesCollection.add(newOpportunityData); // Write the opportunity to Firestore
    const docSnap = await docRef.get();
    const opportunity = docSnap.data() as Opportunity;

    console.log('Services: createOpportunity - Opportunity created successfully with ID:', docRef.id);

    // Step 7: Remove the `id` field from Firestore data to avoid overwriting ID
    const { id: firestoreId, ...opportunityWithoutId } = opportunity;

    // Step 8: Return the newly created opportunity with its autogenerated ID
    return {
      id: docRef.id,
      ...opportunityWithoutId, // Spread other fields without overwriting the autogenerated ID
    };
  } catch (error) {
    console.error('Services: createOpportunity - Error creating opportunity:', error);
    throw error; // Propagate the error to the caller
  }
}

Delete Opportunity‚Äã

import { getOpportunitiesCollection } from '@/lib/firestore-collections';

export async function deleteOpportunity(opportunityId: string): Promise<boolean> {
  const opportunitiesCollection = getOpportunitiesCollection();
  try {
    await opportunitiesCollection.doc(opportunityId).delete();

Get Confidential Opportunities‚Äã

import { getAdminDb } from '@/lib/firebase-admin-server';
import { Opportunity } from '@/features/opportunities/types';
import { handleFirestoreError } from '@/lib/errorHandler';
import { UserRole } from '@/features/auth/types';

/**
 * Fetch all confidential opportunities, restricted to admin or confidential roles.
 * @param userRole The role of the user requesting the confidential opportunities.
 * @returns An array of confidential opportunities.
 */
export async function getConfidentialOpportunities(userRole: UserRole): Promise<Opportunity[]> {
  try {
    console.log('Services: getConfidentialOpportunities - Starting with user role:', userRole);

    // Restrict access to admin or confidential users only
    if (userRole !== 'admin' && userRole !== 'confidential') {
      console.warn('Services: getConfidentialOpportunities - Access denied for user role:', userRole);
      return []; // Return an empty array or handle unauthorized access
    }

    // Initialize Firestore
    const adminDb = await getAdminDb();
    const opportunitiesCol = adminDb.collection('opportunities');

    // Query confidential opportunities from Firestore
    const querySnapshot = await opportunitiesCol
      .where('isConfidential', '==', true)
      .get();

    // Map results to Opportunity objects
    const opportunities = querySnapshot.docs.map((doc) => ({
      id: doc.id,
      ...doc.data(),
    })) as Opportunity[];

    console.log('Services: getConfidentialOpportunities - Retrieved:', opportunities.length, 'opportunities');

    return opportunities;
  } catch (error) {
    console.error('Services: getConfidentialOpportunities - Error fetching opportunities:', error);
    throw handleFirestoreError(error);
  }
}

Get Opportunities‚Äã

import { getAdminDb } from '@/lib/firebase-admin-server';
import {
  FirestoreDataConverter,
  QuerySnapshot,
  Query,
  Filter, // Importing from firebase-admin
} from 'firebase-admin/firestore';
import { Opportunity, OpportunityType, OpportunityVisibility } from '@/features/opportunities/types';
import { UserRole } from '@/features/auth/types';
import { opportunityConverter } from '@/lib/converters/opportunity-converter';
import { handleFirestoreError } from '@/lib/errorHandler';

/**
 * Fetch a paginated list of opportunities based on user role.
 */
export async function getOpportunities(
  userRole: UserRole = UserRole.SUBSCRIBER,
  limit: number = 20,
  startAfter?: string
): Promise<{ opportunities: Opportunity[]; lastVisible: string | null }> {
  try {
    console.log('Services: getOpportunities - Starting...');

    // Access Firestore and initialize collection with converter
    const adminDb = await getAdminDb();
    const opportunitiesCol = adminDb
      .collection('opportunities')
      .withConverter(opportunityConverter as FirestoreDataConverter<Opportunity>);

    // Build the query based on user role and apply filters
    let fireQuery: Query<Opportunity> = opportunitiesCol.orderBy('dateCreated', 'desc').limit(limit);

    if (userRole !== 'admin' && userRole !== 'confidential') {
      // Apply role-based visibility filtering with `Filter`
      fireQuery = fireQuery.where(
        Filter.or(
          Filter.where('visibility', '==', 'public'),
          Filter.where('visibility', '==', 'subscriber'),
          Filter.where('visibility', '==', userRole)
        )
      );
    }

    // Handle pagination using `startAfter`
    if (startAfter) {
      const startAfterDoc = await opportunitiesCol.doc(startAfter).get();
      if (startAfterDoc.exists) {
        fireQuery = fireQuery.startAfter(startAfterDoc);
      }
    }

    // Execute the query and process results
    const querySnapshot: QuerySnapshot<Opportunity> = await fireQuery.get();

    // Map document snapshots to Opportunity objects
    const opportunities = querySnapshot.docs.map((doc) => ({
      id: doc.id, // Explicitly add the ID
      ...doc.data(), // Spread all other document fields
    }));

    // Get the ID of the last visible document for pagination
    const lastVisible =
      querySnapshot.docs.length > 0 ? querySnapshot.docs[querySnapshot.docs.length - 1].id : null;

    console.log('Services: getOpportunities - Retrieved:', opportunities.length, 'opportunities');
    return { opportunities, lastVisible };
  } catch (error) {
    console.error('Services: getOpportunities - Error fetching opportunities:', error);
    throw handleFirestoreError(error); // Handle Firestore-specific errors
  }
}

Get Opportunity By ID‚Äã

import { getAdminDb } from '@/lib/firebase-admin-server';
import { Opportunity, OpportunityType, OpportunityVisibility } from '@/features/opportunities/types';
import { handleFirestoreError } from '@/lib/errorHandler';

/**
 * Fetch an opportunity by its ID.
 */
export async function getOpportunityById(id: string): Promise<Opportunity | null> {
  try {
    console.log('Services: getOpportunityById - Fetching opportunity with ID:', id);

    const adminDb = await getAdminDb();
    const docRef = adminDb.collection('opportunities').doc(id);
    const docSnap = await docRef.get();

    if (!docSnap.exists) {
      console.warn('Services: getOpportunityById - No opportunity found with ID:', id);
      return null; // Return null if the opportunity does not exist
    }

    const data = docSnap.data();
    if (data) {
      const opportunity: Opportunity = {
        id: docSnap.id, // Add document ID explicitly
        ...data,
        type: data.type as OpportunityType,
        visibility: data.visibility as OpportunityVisibility,
      };
      console.log('Services: getOpportunityById - Opportunity retrieved successfully');
      return opportunity;
    }

    return null;
  } catch (error) {
    console.error('Services: getOpportunityById - Error fetching opportunity:', error);
    throw handleFirestoreError(error); // Handle Firestore errors with meaningful responses
  }
}

Get Opportunities By Slug‚Äã

import { getAdminDb } from '@/lib/firebase-admin-server';
import { Opportunity } from '@/features/opportunities/types';
import { handleFirestoreError } from '@/lib/errorHandler';

/**
 * Fetch opportunities that match specified tags (via slug array).
 */
export async function getOpportunitiesBySlug(slugs: string[]): Promise<Opportunity[]> {
  try {
    console.log('Services: getOpportunitiesBySlug - Starting with slugs:', slugs);

    const adminDb = await getAdminDb();
    const opportunitiesCol = adminDb.collection('opportunities');

    // Create query for matching tags
    const slugQuery = slugs.length
      ? opportunitiesCol.where('tags', 'array-contains-any', slugs)
      : opportunitiesCol;

    const opportunitySnapshot = await slugQuery.get();

    // Map and return resulting opportunities
    const opportunities = opportunitySnapshot.docs.map((doc) => ({
      id: doc.id,
      ...doc.data(),
    })) as Opportunity[];

    console.log(
      'Services: getOpportunitiesBySlug - Retrieved:',
      opportunities.length,
      'opportunities'
    );

    return opportunities;
  } catch (error) {
    console.error('Services: getOpportunitiesBySlug - Error fetching opportunities by slug:', error);
    throw handleFirestoreError(error); // Handle Firestore-specific errors
  }
}

Error Responses‚Äã

Both getOpportunities and getConfidentialOpportunities respond with the following errors:

  • 403 Forbidden: If users without required roles attempt to access restricted opportunities.
  • 500 Internal Server Error: For unexpected failures in fetching data from Firestore.

Best Practices‚Äã

  • Use Firestore indexes for filtering and efficiency:

    {
      fields: [{ fieldPath: 'isConfidential', order: 'ASCENDING' }]
    }

  • Leverage pagination for large data sets:

    • Use startAfter with the id of documents instead of unnecessary client-side load.
