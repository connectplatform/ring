# Email AI-CRM
<Callout type="success" title="Executive summary">
Ring ships an **AI-assisted email CRM** for public contact mailboxes (for example `info@example.com`). Inbound mail is classified, threaded, and answered with Claude drafts; platform admins review, approve, and send from the admin UI. Ingest is **IMAP cron poll** or an **inbound webhook**; persistence is PostgreSQL JSONB when migrations `009` + `010` are applied.
</Callout>

## Capability overview

| Layer | Responsibility |
|-------|----------------|
| **Ingestion** | `POST /api/cron/email-processor` (`action: poll`) or `POST /api/webhooks/email/inbound` |
| **Processor** | `services/email/email-processor.ts` — parse, security, AI, CRM, drafts |
| **Persistence** | PostgreSQL JSONB (`email_threads`, `email_contacts`, `email_messages`, `email_drafts`, `email_tasks`, `email_api_usage`) |
| **Outbound** | `services/email/smtp/email-sender.ts` (nodemailer) — replies with `In-Reply-To` / `References` |
| **Admin UI** | `/admin/email-inbox`, drafts, contacts, analytics, tasks, thread detail |
| **Admin API** | `/api/admin/email/*` — `auth()` + `isPlatformAdmin` on every route |

### Business rules

- **Message-ID dedup** — the same RFC `Message-ID` is never processed twice (avoids duplicate llm spend).
- **Auto-send off by default** — set `EMAIL_AUTO_SEND_ENABLED=true` only after an SMTP smoke test passes.
- **Cron poll in serverless** — production schedules `poll` every 1–5 minutes; `EMAIL_PROCESSOR_AUTOSTART` is for long-lived Node workers only.
- **Admin-only** — all `/api/admin/email/*` routes require a platform admin session.

### Admin pages

| Route | Purpose |
|-------|---------|
| `/admin/email-inbox` | Thread list (live API) |
| `/admin/email-inbox/[id]` | Thread detail + messages |
| `/admin/email-drafts` | Pending AI drafts — approve / reject / send |
| `/admin/email-contacts` | CRM contacts registry |
| `/admin/email-analytics` | Volume, intent/sentiment, API cost |
| `/admin/email-tasks` | Follow-ups and escalations |

Navigation: **Admin → Email Inbox / Drafts / Tasks** (right-rail quick nav in `AdminWrapper`).

### Operator setup

<Steps>

<Step>

Copy the **EMAIL CRM** block from `env.local.template` into `.env.local`. Set `IMAP_*`, `SMTP_*`, `ANTHROPIC_API_KEY`, and `CRON_SECRET`. Keep `EMAIL_AUTO_SEND_ENABLED=false` until verified.

</Step>

<Step>

Apply JSONB migrations on Postgres (`009_email_crm_jsonb.sql` + `010_email_crm_tasks_jsonb.sql`). For a quick UI-only test without tables, set `EMAIL_CRM_PERSISTENCE=memory` (data does not survive restarts).

</Step>

<Step>

Trigger ingest: schedule `POST /api/cron/email-processor` with `{"action":"poll"}` and `Authorization: Bearer $CRON_SECRET` every 1–5 minutes — or use the inbound webhook (see [developer guide](/docs/examples/email-ai-crm)).

</Step>

<Step>

Open **Admin → Email Inbox** and **Email Drafts**. Approve a draft and send. Confirm the reply in the sender mailbox before enabling auto-send.

</Step>

</Steps>

<Tab value="Developers">

### Processing pipeline

1. **Parse** — headers, body, attachments, thread keys (`References`, `In-Reply-To`).
2. **Security** — four-layer prompt-injection defense (`services/email/security/`).
3. **AI** — Haiku intent + sentiment; Sonnet response generation with prompt caching.
4. **CRM** — contact upsert, thread upsert, optional auto-tasks.
5. **Draft** — pending queue unless auto-send criteria are met.
6. **Notify** — `EmailNotificationService` (in-app / optional Slack) on draft, task, security block, urgent mail.

### Code layout

| Path | Role |
|------|------|
| `services/email/email-processor.ts` | Orchestrator; dedup, poll, ingest |
| `services/email/imap/` | Config, listener, `pollBatch()` |
| `services/email/smtp/email-sender.ts` | Outbound replies |
| `services/email/crm/` | Contact + task services |
| `services/email/drafts/` | Draft queue + auto-send rules |
| `services/email/ai/` | Classifiers, generator, cost tracker |
| `features/email-crm/services/` | Thread, message, analytics |
| `features/email-crm/repositories/` | JSONB repositories |
| `app/api/cron/email-processor/route.ts` | Cron: `poll`, `status`, `mark-overdue-tasks` |
| `app/api/webhooks/email/inbound/route.ts` | HMAC-signed webhook ingest |

### Cron actions

`POST /api/cron/email-processor` accepts `action`: `poll` (default), `status`, `mark-overdue-tasks`, and optionally `start` / `stop` when `EMAIL_PROCESSOR_ALLOW_HTTP_START=true`.

## Local configuration

Add to **`.env.local`** (canonical comments in `env.local.template` **EMAIL CRM** block):

<Code language="env" title=".env.local — Email AI-CRM essentials">
{`IMAP_HOST=mail.example.com
IMAP_PORT=993
IMAP_TLS=true
IMAP_USER=info@example.com
IMAP_PASSWORD=

SMTP_HOST=mail.example.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=info@example.com
SMTP_PASSWORD=

ANTHROPIC_API_KEY=

CRON_SECRET=generate-a-long-random-string
EMAIL_AUTO_SEND_ENABLED=false
EMAIL_CRM_PERSISTENCE=memory
WEBHOOK_EMAIL_SECRET=`}
</Code>

| Variable | Purpose |
|----------|---------|
| `IMAP_*` / `SMTP_*` | Community inbox (any IMAP/SMTP provider) |
| `ANTHROPIC_API_KEY` | Intent, sentiment, and draft generation |
| `CRON_SECRET` | Bearer token for `/api/cron/email-processor` |
| `EMAIL_AUTO_SEND_ENABLED` | Keep `false` until SMTP smoke test passes |
| `WEBHOOK_EMAIL_SECRET` | HMAC for `POST /api/webhooks/email/inbound` (optional alternative to IMAP) |
| `EMAIL_CRM_PERSISTENCE` | `memory` for quick local UI tests; omit when using migrations `009`/`010` |
| `EMAIL_POLLING_INTERVAL` | Poll interval in ms (default `30000`) |
| `EMAIL_PROCESSOR_AUTOSTART` | Long-lived worker IDLE mode — not for serverless cron |

<Callout type="warning" title="Deprecated migration">
Do **not** apply `001_email_crm_schema.sql` (column-model CRM). Use **`009` + `010` JSONB migrations** only. See [developer guide](/docs/examples/email-ai-crm).
</Callout>

## Production deployment

On k8s or Vercel cron, store the same variables in your secret manager, apply migrations `009` + `010`, and schedule:

<Code language="bash" title="Production cron poll (every 1–5 min)">
{`curl -sS -X POST "$BASE_URL/api/cron/email-processor" \\
  -H "Authorization: Bearer ${CRON_SECRET}" \\
  -H "Content-Type: application/json" \\
  -d '{"action":"poll"}'`}
</Code>

Optional analytics snapshot: `GET $BASE_URL/api/cron/email-analytics` with the same Bearer token.
