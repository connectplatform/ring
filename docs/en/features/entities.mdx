## 🚀 Implementation & Integration

Production clones use **PostgreSQL via `DatabaseService`**. Server reads return **`SerializedEntity`** (ISO date strings). Post-mutation freshness uses **`syncEntityDiscovery()`** — cache tags, `revalidatePath`, and Tunnel `entity:*` events (see [Discovery mutation sync](../architecture/discovery-mutation-sync)).

### Data layer

| Path | Module |
|------|--------|
| PostgreSQL read/write | `features/entities/lib/entity-db-mapper.ts` |
| Post-mutation sync | `features/entities/lib/entity-mutation-sync.ts` |
| Shared Tunnel publish | `lib/discovery/sync-discovery.ts` |
| My Entities route | `/[locale]/entities/my` — `getMyEntities()` with owned / store / member tabs |
| Firestore legacy | `lib/converters/entity-converter.ts` (`firebase-full` only) |
