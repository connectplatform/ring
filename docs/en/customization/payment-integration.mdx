## Payment purposes

| Purpose | Typical use | Handler |
|---------|-------------|---------|
| `store_order` | Multi-vendor store checkout | `handlers/store-order.ts` |
| `membership_upgrade` | SUBSCRIBER вЖТ MEMBER | `handlers/membership-upgrade.ts` |
| `news_promotion` | News Kingdom paid promotion | `handlers/news-promotion.ts` |
| `wallet_topup` | Wallet credit top-up | conductor + wallet services |

**Processors:** WayForPay (default), Stripe, internal credit. PayPal is **not** in the OSS tree.

**Rails:** `merchant_redirect` ¬Ј `internal_credit` (`PAYMENT_STORE_ALLOW_CREDIT`) ¬Ј `native_token` (`PAYMENT_STORE_ALLOW_TOKEN=true`).

Apply `data/migrations/004_payment_transactions.sql` before production payments.

## Environment variables

<Code language="bash" title=".env.local вАФ PaymentConductor">
{`PAYMENT_DEFAULT_PROCESSOR=wayforpay
PAYMENT_STORE_PROCESSOR=wayforpay
PAYMENT_MEMBERSHIP_PROCESSOR=wayforpay
PAYMENT_NEWS_PROCESSOR=wayforpay
WAYFORPAY_MERCHANT_ACCOUNT=your_merchant
WAYFORPAY_SECRET_KEY=your_secret
WAYFORPAY_DOMAIN=yourdomain.com
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
PAYMENT_FIAT_CURRENCY=UAH
PAYMENT_STORE_ALLOW_CREDIT=true
PAYMENT_STORE_ALLOW_TOKEN=false`}
</Code>

Full commented block: [Environment вАФ PaymentConductor](/docs/deployment/environment#payment-conductor).

## Webhook endpoints

<Tabs items={['Operators', 'Developers']}>

<Tab value="Operators">

Register **canonical** URLs in your PSP merchant cabinet (replace `yourdomain.com`):

| Provider | Service URL / endpoint |
|----------|------------------------|
| WayForPay | `https://yourdomain.com/api/payments/wayforpay/webhook` |
| Stripe | `https://yourdomain.com/api/payments/stripe/webhook` |

<Callout type="error" title="Do not register">
  Legacy paths `/api/payments/wayforpay/callback`, `/success`, `/failure` вАФ **not implemented** in v1.6.
</Callout>

Deprecated aliases (still work вАФ migrate when convenient):

- `/api/store/payments/wayforpay/webhook`
- `/api/news/promotion/wayforpay-webhook`

</Tab>

<Tab value="Developers">

| Route file | Method | Role |
|------------|--------|------|
| `app/api/payments/wayforpay/webhook/route.ts` | POST | `dispatchWayForPayWebhook` |
| `app/api/payments/stripe/webhook/route.ts` | POST | `dispatchStripeWebhook` |
| `app/api/store/payments/wayforpay/route.ts` | POST | Initiate store redirect |
| `app/api/store/payments/credit/route.ts` | POST | Internal credit checkout |
| `app/api/store/payments/[orderId]/status/route.ts` | GET | Poll order status |

WayForPay routes by `orderReference` prefix: `store_`, `membership_`, `news-promo-вА¶` вАФ see `lib/payments/conductor/webhook-dispatcher.ts`.

</Tab>

</Tabs>

## PSP setup

<Tabs items={['WayForPay', 'Stripe']}>

<Tab value="WayForPay">

<Callout type="success">
  **Recommended for:** Ukraine / Europe вАФ hosted cards, Apple Pay, Google Pay.
</Callout>

<Steps>

<Step>

Create a merchant account at [wayforpay.com](https://wayforpay.com) вАФ complete KYC and note **merchant account**, **secret key**, approved **domain**.

</Step>

<Step>

Set Ring env vars:

<Code language="bash" title="WayForPay">
{`WAYFORPAY_MERCHANT_ACCOUNT=your_merchant
WAYFORPAY_SECRET_KEY=your_secret
WAYFORPAY_DOMAIN=yourdomain.com`}
</Code>

</Step>

<Step>

In the WayForPay cabinet, set **service URL** (callback):

<Code language="text" title="Service URL">
{`https://yourdomain.com/api/payments/wayforpay/webhook`}
</Code>

</Step>

<Step>

Verify in staging: migration `004` applied вЖТ test checkout вЖТ confirm `payment_transactions` row + fulfillment (order paid / role upgraded / promotion active).

</Step>

</Steps>

</Tab>

<Tab value="Stripe">

<Steps>

<Step>

Create account at [stripe.com](https://stripe.com); obtain test or live API keys.

</Step>

<Step>

<Code language="bash" title="Stripe env">
{`PAYMENT_DEFAULT_PROCESSOR=stripe
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...`}
</Code>

</Step>

<Step>

Stripe Dashboard вЖТ Webhooks вЖТ add endpoint:

<Code language="text" title="Endpoint URL">
{`https://yourdomain.com/api/payments/stripe/webhook`}
</Code>

Subscribe to `checkout.session.completed` (and events your processor handles). Copy signing secret to `STRIPE_WEBHOOK_SECRET`.

</Step>

</Steps>

</Tab>

</Tabs>

## Pre-launch checklist

<Steps>

<Step>

Migration `004_payment_transactions.sql` applied on production Postgres.

</Step>

<Step>

`PAYMENT_*` and PSP credentials in secrets manager (never commit `.env.local`).

</Step>

<Step>

Canonical webhook URLs registered; `WAYFORPAY_DOMAIN` matches approved merchant domain.

</Step>

<Step>

Staging smoke: store checkout, membership upgrade, news promotion (if used).

</Step>

<Step>

Post go-live: monitor `payment_transactions` for stuck `pending` / `redirected` rows.

</Step>

</Steps>
