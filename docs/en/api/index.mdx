# API Reference

Ring Platform exposes **132 route handlers** across entities, opportunities, store, payments (PaymentConductor), news, wallet, tunnel, MCP, and more (v1.6.0).

<Callout type="info" title="Authentication model">
**Browser / same-app clients** — Auth.js v5 **session cookie** on same-origin `fetch` (`credentials: 'include'`). No manual `Bearer` header in React client code.

**Automation** — `Authorization: Bearer` for cron (`CRON_SECRET`), webhooks, email ingest, and **`/api/mcp/v1/*`** (`RING_MCP_SERVICE_TOKENS`). See [Ring MCP](/en/docs/development/ring-mcp).
</Callout>

## Quick start

### Base URL

Use your clone origin — local dev:

```
http://localhost:3000/api
```

White-label examples: [connect.software](https://connect.software) (A2B MCP Marketplace), [ring-platform.org](https://ring-platform.org) (Ring community portal).

### Example (session cookie)

Sign in via the browser, then:

<Code language="bash" title="terminal">
{`curl -s "http://localhost:3000/api/entities?limit=5" \\
  -H "Cookie: authjs.session-token=YOUR_SESSION_COOKIE"`}
</Code>

Integration patterns: [API integration examples](/en/docs/examples/api-integration).

## Realtime

In-app live updates use **Tunnel Protocol** (native WSS on k8s, SSE/poll on Vercel). See [Tunnel protocol](/en/docs/features/tunnel-protocol) and [Realtime architecture](/en/docs/architecture/real-time).

## Security

- Auth.js v5 sessions (cookie in browser; JWT strategy server-side)
- Role-based access control (VISITOR → ADMIN)
- Rate limiting on sensitive routes (e.g. notifications API)
- Webhook/cron/MCP routes use explicit Bearer secrets
