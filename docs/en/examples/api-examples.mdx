# API Examples

Quick cURL samples for manual testing. For TypeScript client patterns, see [API integration examples](/en/docs/examples/api-integration).

<Callout type="info" title="Auth">
Route handlers use **Auth.js session cookies**. Sign in through the UI, copy `authjs.session-token` from devtools, or use same-origin `fetch` from client code (cookie sent automatically).
</Callout>

## Entities

<Code language="bash" title="terminal">
{`# List entities
curl -s "http://localhost:3000/api/entities?limit=10" \\
  -H "Cookie: authjs.session-token=YOUR_SESSION_COOKIE"

# Create entity
curl -s -X POST "http://localhost:3000/api/entities/create" \\
  -H "Cookie: authjs.session-token=YOUR_SESSION_COOKIE" \\
  -H "Content-Type: application/json" \\
  -d '{"name":"Acme","type":"technologySoftware","shortDescription":"…"}'

# Get one entity
curl -s "http://localhost:3000/api/entities/ENTITY_ID" \\
  -H "Cookie: authjs.session-token=YOUR_SESSION_COOKIE"

# Delete entity (requires confirm in body)
curl -s -X DELETE "http://localhost:3000/api/entities/ENTITY_ID" \\
  -H "Cookie: authjs.session-token=YOUR_SESSION_COOKIE" \\
  -H "Content-Type: application/json" \\
  -d '{"confirm":true}'`}
</Code>

## Opportunities

<Code language="bash" title="terminal">
{`curl -s "http://localhost:3000/api/opportunities?limit=10" \\
  -H "Cookie: authjs.session-token=YOUR_SESSION_COOKIE"`}
</Code>

Create and update opportunities in the app UI use **Server Actions** (`app/_actions/opportunities.ts`), not `POST /api/opportunities`.

curl -s "http://localhost:3000/api/conversations/CONVERSATION_ID/messages?limit=20" \\

## MCP (Bearer token)

<Code language="bash" title="terminal">
{`curl -s "http://localhost:3000/api/mcp/v1/entities?limit=5" \\
  -H "Authorization: Bearer YOUR_RING_MCP_TOKEN"`}
</Code>

See [Ring MCP](/en/docs/development/ring-mcp) for token configuration.
