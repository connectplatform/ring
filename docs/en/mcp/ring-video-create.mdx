# `ring-video-create`

Generate a short video from a text prompt (or image for I2V), poll xAI asynchronously, optionally upload to **ring-filebase**, and return CDN + temporary URLs.

<Callout type="success" title="Executive summary">
Operators call **`ring-video-create`** through `ring-mcp`. The gateway hits `POST /api/mcp/v1/videos/generate`, which runs **`VideoConductor`** (`lib/video/conductor/video-conductor.ts`). Default **`qualityMode: draft`** uses **`grok-imagine-video` @ 480p** (~**$0.05/second**) ‚Äî cheapest text-to-video iteration. Set **`remaster: true`** or **`qualityMode: production`** for **720p** finals.
</Callout>

## Tool identity

| Field | Value |
|-------|--------|
| **MCP tool name** | `ring-video-create` |
| **HTTP route** | `POST /api/mcp/v1/videos/generate` |
| **MCP server** | `AI-RING/ring-mcp/lib/tools.js` |
| **Gateway handler** | `app/api/mcp/v1/videos/generate/route.ts` |
| **Conductor** | `lib/video/conductor/video-conductor.ts` |
| **Presets SSOT** | `lib/video/video-presets.json` |
| **Confirm required** | No |

## Quality modes & cost

| `qualityMode` | Model | Resolution | ~$/sec | Text-to-video |
|---------------|--------|------------|--------|---------------|
| **`draft`** (default) | `grok-imagine-video` | **480p** | **$0.05** | Yes |
| **`production`** | `grok-imagine-video` | **720p** | $0.05 | Yes |
| **`production_i2v`** | `grok-imagine-video-1.5` | 720p | $0.14 | **No** ‚Äî needs `imageUrl` |

**Recommended workflow:** draft iterations ‚Üí `remaster: true` with same `prompt` + `remasterFromRequestId` from draft.

## Parameters

| Parameter | Required | Type | Notes |
|-----------|----------|------|-------|
| `prompt` | **Yes** | string | Scene / motion description |
| `qualityMode` | No | `draft` \| `production` \| `production_i2v` | Default `draft` |
| `remaster` | No | boolean | Forces production 720p preset |
| `duration` | No | number | 1‚Äì15 seconds (default 6) |
| `aspectRatio` | No | string | e.g. `16:9`, `9:16` |
| `resolution` | No | `480p` \| `720p` | Override preset |
| `model` | No | string | Override xAI model slug |
| `imageUrl` | No | string | Starting frame for I2V / 1.5 |
| `purpose` | No | string | Storage path segment |
| `refCode` | No | string | Tracking id |
| `remasterFromRequestId` | No | string | Links draft job in audit |
| `persistToFilebase` | No | boolean | Default true on MCP |

## Examples

**Draft (cheap iteration):**

```
ring-video-create prompt "HOA neighbors on video call, frustrated, documentary realism" qualityMode draft duration 6 purpose viral-ad
```

**Production remaster:**

```
ring-video-create prompt "‚Ä¶same locked prompt‚Ä¶" remaster true remasterFromRequestId <draft-request-id> qualityMode production duration 10
```

## CLI batch runner (operators)

For multi-clip campaigns, use the cumulative manifest script:

```bash
cd ring-platform.org/scripts/ring-viral-video
node generate-xai-videos.mjs --draft --download
node generate-xai-videos.mjs --remaster 01_hoa_video_call_pains --download
```

Manifest upserts by `id::qualityMode` ‚Äî partial runs do not wipe prior clips.

## Troubleshooting

| Symptom | Check |
|---------|--------|
| Timeout | Raise `VIDEO_GEN_POLL_TIMEOUT_MS` (video jobs take minutes) |
| `does not support text-to-video` | Use `grok-imagine-video` (draft/production), not 1.5 without `imageUrl` |
| Moderation filtered | `respectModeration: false` in response ‚Äî revise prompt |
| Expired temporary URL | Re-poll or use `persistToFilebase` CDN `url` |
