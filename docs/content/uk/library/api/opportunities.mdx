# Opportunities API

Управління двоприродним маркетплейсом можливостей з **7 ендпоінтами**, що підтримують як пропозиції, так і запити для різних типів можливостей.

## Огляд

Система можливостей Ring Platform підтримує унікальний двоприродний підхід:
- **Пропозиції**: Що надають організації (робота, партнерство, ресурси)
- **Запити**: Що потрібно організаціям (послуги, партнерство, ресурси)

**7 типів можливостей:**
- Робочі можливості
- Партнерські можливості
- Волонтерські можливості
- Менторські можливості
- Ресурсні можливості
- Подієві можливості
- Користувацькі можливості

## API Ендпоінти

### `GET /api/opportunities`
Список можливостей з розширеною фільтрацією

### `POST /api/opportunities`
Створити нову можливість (потрібна роль MEMBER)

### `GET /api/opportunities/{id}`
Отримати деталі можливості за ID

### `PUT /api/opportunities/{id}`
Оновити інформацію про можливість

### `DELETE /api/opportunities/{id}`
Видалити можливість

### `POST /api/opportunities/{id}/apply`
Подати заявку на можливість або виразити зацікавленість

### `GET /api/opportunities/{id}/applications`
Список заявок на можливість

## Контроль Доступу

### Видимість Можливостей
- **Публічні Можливості**: Видимі всім користувачам
- **Конфіденційні Можливості**: Потребують роль CONFIDENTIAL

### Управління Можливостями
- **Створення**: Потрібна роль MEMBER
- **Оновлення/Видалення**: Створювач можливості або адміністратор організації
- **Подача заявки**: Будь-який авторизований користувач (залежно від ролі)

### Управління Заявками
- **Перегляд Заявок**: Створювач можливості та адміністратори організації
- **Управління Заявками**: Прийняття/відхилення заявок

## Розширені Функції

### Управління Дедлайнами
- Автоматичне закінчення терміну можливостей
