# Компоненти документації

Документація Ring — це **MDX-файли**, які рендерить `next-mdx-remote/rsc` зі спільною мапою компонентів у `components/docs/mdx-docs-shared.tsx`. Використовуйте ці блоки для сторінок, зрозумілих **розробникам** (шляхи, код) та **керівникам** (результат, одна діаграма, чітка навігація).

<Callout type="info">
  **Канонічні шляхи:** контент у `docs/content/{locale}/library/{section}/{page}.mdx`. Публічні URL без `library` — наприклад `/docs/architecture`, `/docs/development/docs-components`.
</Callout>

## Конвеєр рендерингу

<Mermaid title="Шлях рендеру доків">
{`flowchart LR
  MDX["MDX + frontmatter"]
  RSC["next-mdx-remote/rsc"]
  GFM["remark-gfm"]
  Rehype["rehype: Code + Mermaid"]
  Map["docsMdxComponents"]
  Page["/{locale}/docs/..."]

  MDX --> RSC
  GFM --> RSC
  Rehype --> RSC
  RSC --> Map --> Page`}
</Mermaid>

| Етап | Розташування | Роль |
|------|--------------|------|
| Контент | `docs/content/{locale}/library/` | `title`, `description`, тіло MDX |
| Resolver | `lib/docs/resolve-doc-file-path.ts` | `/docs/foo` → `library/foo.mdx` |
| Компоненти | `components/docs/mdx-docs-shared.tsx` | Реєстрація JSX-тегів |
| Підсвітка | Shiki (`nord` / `tokyo-night`) | Серверний `<Code>` |

## Швидкий довідник компонентів

| Компонент | Найкраще для | Client / server |
|-----------|--------------|-----------------|
| **Callout** | Резюме для керівників, попередження (`DB_BACKEND_MODE`, legal) | Client |
| **Mermaid** | Одна системна карта + одна sequence на хаб (без розростання) | Client |
| **Tabs / Tab** | Розробник vs Оператор (або CEO) на одній сторінці | Client |
| **Cards / Card** | Навігація хаба до дочірніх статей | Server-friendly |
| **Steps / Step** | Інсталяція та деплой покроково | Server-friendly |
| **Code** | Конфіг з Shiki | Server (async) |
| **MindMap** | Дерева понять — обережно на хабах | Client |
| **Timeline** | Історія релізів / міграцій | Client |
| **RingAISynapseFlow** | Маркетинг / AI-демо — важкий бандл | Client |
| **CodeSandbox** | Живі приклади в `/examples` | Client |
| **Math / MathBlock** | Науковий редактор, токеноміка | Client |

Огороджені блоки \`\`\`typescript та \`\`\`mermaid перетворюються на `<Code>` та `<Mermaid>` через rehype.

## Callout

Типи: `info` | `warning` | `error` | `success`. Опційно `title`.

<Callout type="success" title="Патерн executive summary">
  Починайте хаби з одного абзацу результату для операторів і CEO, далі таблиця та одна системна діаграма.
</Callout>

## Mermaid

На хабах — **одна** архітектурна карта та **одна** sequence. Решта — на дочірніх сторінках.

## Steps і Step

<Callout type="warning" title="Синтаксис MDX">
  **Потрібні порожні рядки** перед `<Steps>`, перед кожним `<Step>`, після `</Step>` і після `</Steps>`.
</Callout>

<Steps>

<Step>

Встановіть залежності:

<Code language="bash" title="terminal">
{`cd ring-platform.org
npm install`}
</Code>

</Step>

<Step>

Додайте сторінку в `meta.json` секції та синхронізуйте UK/RU за `LOCALE-GAPS.md`.

</Step>

<Step>

Перевірте `http://localhost:3000/docs/{section}/{page}`.

</Step>

</Steps>

## Правила авторингу

| Правило | Чому |
|---------|------|
| EN канонічний, UK/RU за LOCALE-GAPS | Без розбіжностей у env/API |
| Frontmatter `title` + `description` | SEO і сайдбар |
| Без `fumadocs-ui` | Лише `components/docs/` |
| `npm run dev` після нового MDX | Ловити помилки Steps/Mermaid |
