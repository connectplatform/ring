# Кастомізація

<Callout type="info">
  **Що це за розділ:** Ring **завжди** white-label — один open-source код, ваш бренд, ваш домен. Ці доки для **операторів і розробників**, які ведуть клон (самостійно або з Ringdom). Це **не** процес замовлення на Ringdom.
</Callout>

## Два способи отримати клон

<Mermaid title="Як ви отримуєте свій Ring">
{`flowchart TD
  Start([Хочу свій Ring])
  Start --> Choice{Хто веде продакшен?}

  Choice -->|Будую / self-host| OSS[OSS на GitHub]
  Choice -->|Ringdom веде за мене| RD[ringdom.org]

  OSS --> A1[./install.sh]
  OSS --> A2[Ручний клон + агент]
  A1 --> Docs[Ці доки — Quick Start далі]
  A2 --> Docs

  RD --> Chat[Чат на головній:\\nОпишіть вашу\\nRing-платформу]
  Chat --> Ops[Ringdom: k8s, DNS, handoff]
  Ops --> Customize[Той самий код — потім Customization]`}
</Mermaid>

| Крок | OSS (ви ведете) | Ringdom (ми ведемо) |
|------|-----------------|---------------------|
| 1 | [Quick Start](/docs/customization/quick-start) | [ringdom.org](https://ringdom.org) — чат на головній |
| 2 | [Customization guide](/docs/customization/customization-guide) | Після handoff — той самий guide |
| 3 | [Self-hosted](/docs/deployment/self-hosted) | Ringdom k8s (не Quick Start) |

<Callout type="warning">
  Для Ringdom-managed settlers **не** використовуйте Quick Start — замовлення починається на [ringdom.org](https://ringdom.org).
</Callout>
