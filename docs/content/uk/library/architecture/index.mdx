## Архітектурні Принципи

### 🎯 **Масштабованість**
- **Horizontal Scaling**: Kubernetes pods можуть масштабуватися автоматично
- **Database Sharding**: PostgreSQL підтримує future sharding для великих обсягів даних
- **CDN Integration**: Vercel Blob для глобального розподілу медіа-контенту

### 🔒 **Безпека**
- **Zero Trust Architecture**: Кожен запит аутентифікується та авторизується
- **Secrets Management**: Kubernetes secrets для всіх sensitive даних
- **SSL/TLS**: End-to-end encryption для всіх з'єднань

### ⚡ **Продуктивність**
- **Edge Runtime**: Next.js Edge функції для швидкого cold start
- **Caching Layers**: React cache, Redis, PostgreSQL query caching
- **Database Optimization**: Connection pooling, query optimization, indexing

### 🔄 **Надійність**
- **Health Checks**: Автоматичне відновлення unhealthy pods
- **Circuit Breakers**: Захист від cascade failures
- **Backup & Recovery**: Automated database backups та disaster recovery

## Режими бекенду та бази даних

Деплой вимагає **`DB_BACKEND_MODE`** (`k8s-postgres-fcm`, `firebase-full` або `supabase-fcm`). У продакшені з пріоритетом Postgres Firebase зазвичай використовується для **FCM**, а не Firestore як основне сховище. Повна матриця, різниця **`DB_HYBRID_MODE`** / режим бекенду, Tunnel Supabase проти `supabase-fcm`: див. **[Режими бекенду та бази даних](/uk/library/architecture/backend-modes-and-databases)**.
