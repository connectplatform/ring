# Email AI-CRM API

> **EN canonical:** повні тіла запитів, приклади curl і JSON webhook — [Email AI-CRM API (EN)](/docs/library/api/email-ai-crm)

<Callout type="warning" title="Доступ адміна">
Усі `/api/admin/email/*` вимагають сесії **platform admin** (`admin` або `superadmin`).
</Callout>

<Callout type="info" title="Cron / webhook">
`/api/cron/*` і `/api/webhooks/email/inbound` — `CRON_SECRET` або `WEBHOOK_EMAIL_SECRET`, не user session.
</Callout>

## Cron — email processor

### `POST` або `GET` `/api/cron/email-processor`

**Auth:** `Authorization: Bearer $CRON_SECRET`

| `action` | Поведінка |
|----------|-----------|
| `poll` (default) | `pollInboundBatch()` — UNSEEN → process → disconnect |
| `status` | Статистика процесора + IMAP |
| `stop` | Зупинити IDLE |
| `start` | IDLE (потрібен `EMAIL_PROCESSOR_ALLOW_HTTP_START=true`) |
| `mark-overdue-tasks` | Прострочені задачі |

```bash
curl -X POST "$BASE_URL/api/cron/email-processor" \
  -H "Authorization: Bearer $CRON_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"action":"poll"}'
```

### `GET /api/cron/email-analytics`

7-денний знімок дашборду (як admin analytics).

## Webhook — вхідна пошта

### `POST /api/webhooks/email/inbound`

**Auth:** `Bearer $WEBHOOK_EMAIL_SECRET` або HMAC-SHA256 у `X-Email-Webhook-Signature`.

Мінімальні поля: `messageId`, `from`, `subject`, `bodyText`. Викликає `EmailProcessor.ingestEvent()` з `uid: 0`.
