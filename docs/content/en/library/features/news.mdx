### Database Schema

The News module uses PostgreSQL with JSONB for flexible content storage:

```sql
CREATE TABLE news (
    id VARCHAR(255) PRIMARY KEY,
    data JSONB NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Performance indexes
CREATE INDEX idx_news_author_id ON news ((data->>'authorId'));
CREATE INDEX idx_news_category ON news ((data->>'category'));
CREATE INDEX idx_news_status ON news ((data->>'status'));
CREATE INDEX idx_news_published_at ON news ((data->>'publishedAt'));
CREATE INDEX idx_news_locale ON news ((data->>'locale'));
CREATE INDEX idx_news_translation_group ON news ((data->>'translationGroupId'));
```

### Service Architecture

All news operations use the `DatabaseService` abstraction layer with React 19 patterns:

- **READ operations**: `cache()` for performance optimization
- **MUTATIONS**: `revalidatePath()` for UI freshness
- **TRANSACTIONS**: PostgreSQL ACID guarantees for data integrity
