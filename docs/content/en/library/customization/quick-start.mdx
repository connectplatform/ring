# Quick Start: Your First Ring Clone

<Callout type="warning" title="OSS self-host only">
  **Ringdom-managed hosting?** Go to **[ringdom.org](https://ringdom.org)** and use the main-page chat — *describe your next Ring-powered platform*. Ringdom provisions production; you do **not** need this Quick Start. After handoff, use the [Customization guide](/docs/customization/customization-guide).
</Callout>

<Callout type="info">
  **Time estimate:** 45–90 minutes for first local smoke test (Postgres + migrations + Auth). **Prerequisites:** Git, Node.js 20+, Docker optional for Postgres.
</Callout>

This guide is for **operators who run the repo themselves**: bash **`install.sh`**, or **manual clone** with agentic assistance (Cursor/LegioX) walking the same steps. Source: [`connectplatform/ring`](https://github.com/connectplatform/ring) on GitHub — free forever.

## What you will have

| Outcome | Details |
|---------|---------|
| Local app | `http://localhost:3000` with marketing home via `HomeWrapper` |
| Backend | `DB_BACKEND_MODE=k8s-postgres-fcm` + PostgreSQL |
| Auth | Auth.js v5 — at minimum `AUTH_SECRET` + one OAuth provider |
| Schema | `data/schema.sql` v4+ and kingdom migrations applied |
| Config | `ring-config.json` + `.env.local` from templates |

## Path 1 — `install.sh` (recommended)

From a fresh directory:

```bash
git clone https://github.com/connectplatform/ring.git
cd ring
chmod +x install.sh
./install.sh --quick
```

`install.sh` v2.0:

- Prompts for clone name and creates **`ring-config.json`** from `ring-config.template.json`
- Copies **`env.local.template`** → `.env.local` and generates `AUTH_SECRET` / wallet keys
- Optionally materializes Docker / `next.config` templates
- Runs `npm install`

Flags: `--clone-name my-ring`, `--quick`, `--help`. See `INSTALL.md` in the repo root.

## Path 2 — Manual setup

<Steps>

<Step>

**Clone and install:**

```bash
git clone https://github.com/connectplatform/ring.git
cd ring
npm install
cp env.local.template .env.local
cp ring-config.template.json ring-config.json
```

Edit `ring-config.json` — at minimum `clone.displayName`, `domains.development`, and `features` toggles.

</Step>

<Step>

**Set required backend mode** in `.env.local`:

```env
DB_BACKEND_MODE=k8s-postgres-fcm
DB_HOST=localhost
DB_PORT=5432
DB_NAME=ring_platform
DB_USER=ring_user
DB_PASSWORD=ring_password_2024
```

<Callout type="warning">
  `DB_BACKEND_MODE` is **required** — the app will not start without it. Do not use legacy `DATABASE_MODE=firebase_only`.
</Callout>

</Step>

<Step>

**PostgreSQL** — pick one:

- **Docker dev** (repo): `docker compose -f docker-compose.dev.yml up -d` (see `DOCKER_SETUP.md`)
- **Homebrew** (Ringdom monorepo): `./infrastructure/postgres/bootstrap-brew-dev.sh` from kingdom root
- **Existing server:** point `DB_*` at your instance

</Step>

<Step>

**Apply schema and migrations:**

```bash
export DATABASE_URL="postgresql://${DB_USER}:${DB_PASSWORD}@${DB_HOST}:${DB_PORT}/${DB_NAME}"
psql "$DATABASE_URL" -f data/schema.sql
# Then kingdom migrations — see full order:
```

Full apply order: [Database migrations](/docs/getting-started/migrations). Helpers: `./scripts/run-migration.sh`, `npm run setup:env`.

</Step>

<Step>

**Auth.js minimum** in `.env.local`:

```env
AUTH_SECRET=   # install.sh generates; or: npx @better-auth/cli secret
AUTH_GOOGLE_ID=
AUTH_GOOGLE_SECRET=
NEXT_PUBLIC_BASE_URL=http://localhost:3000
```

Optional: `AUTH_APPLE_*`, magic link, Web3 — see [Authentication](/docs/architecture/authentication).

</Step>

<Step>

**Locales** (defaults are fine for first run):

```env
NEXT_PUBLIC_SUPPORTED_LOCALES=en,uk,ru
NEXT_PUBLIC_DEFAULT_LOCALE=en
```

SSOT: `lib/locale-config.ts`.

</Step>

<Step>

**Start dev server:**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). The home route renders **`components/wrappers/home-wrapper.tsx`** (marketing shell + right rail), not a separate portal app.

</Step>

</Steps>

## Smoke-test checklist

<Tabs items={['Developers', 'Operators']}>

<Tab value="Developers">

- [ ] Home loads — hero from `locales/{locale}/pages.json` → `pages.home`
- [ ] `/login` — Google OAuth or configured provider
- [ ] `/entities` and `/opportunities` — DB reads succeed (no adapter init errors in terminal)
- [ ] `npm run type-check` passes
- [ ] Optional: `npm run smoke:pipelines` after store/payment env is set

</Tab>

<Tab value="Operators">

- [ ] `DB_BACKEND_MODE` documented in runbook
- [ ] Migrations version matches `data/migrations/` list
- [ ] FCM vars present only if push is enabled (Firebase ≠ primary DB in Postgres mode)
- [ ] Production secrets in k8s / secret manager — never commit `.env.local`
- [ ] Ingress + `NEXT_PUBLIC_BASE_URL` aligned with public domain

</Tab>

</Tabs>

## Production deployment (outline)

OSS self-host does **not** ship empire k8s manifests. Recommended path:

1. [Self-hosted deployment](/docs/deployment/self-hosted) — Docker or `npm run build && npm start`
2. [Environment configuration](/docs/deployment/environment) — PaymentConductor, FCM, WayForPay/Stripe
3. **Ringdom settlers** — order at [ringdom.org](https://ringdom.org) via the homepage chat (*describe your next Ring-powered platform*); skip this Quick Start

<Callout type="warning">
  **Not recommended as primary production data plane:** Vercel serverless + Firestore-only (`firebase-full`) for full Ring v1.6 feature surface. Postgres-primary on your cluster is the production default.
</Callout>
