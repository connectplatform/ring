# Data Model

Core collections and relationships (Firestore):

- `users` ↔ profiles, wallets
- `entities` ↔ owners (users), verification, visibility tiers
- `opportunities` ↔ entities, applicants
- `notifications` ↔ users


<Callout type="warning">Deploy required indexes per Firestore config.</Callout>
