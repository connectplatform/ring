# Email AI-CRM — гайд разработчика

Настройка модуля на **локальном или self-hosted** клоне Ring: `.env.local`, миграции (при необходимости), проверка ingest и отправки черновика из админ UI.

> **EN canonical:** полный troubleshooting и webhook — [Email AI-CRM developer guide (EN)](/docs/library/examples/email-ai-crm)

## 1. Локальное окружение

Блок **EMAIL CRM** из `env.local.template` → `.env.local`:

```bash
# .env.local
IMAP_HOST=mail.example.com
IMAP_USER=info@example.com
IMAP_PASSWORD=

SMTP_HOST=mail.example.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=info@example.com
SMTP_PASSWORD=

ANTHROPIC_API_KEY=

CRON_SECRET=generate-a-long-random-string
EMAIL_AUTO_SEND_ENABLED=false
EMAIL_CRM_PERSISTENCE=memory
```

## 2. Миграции

```bash
psql "$DATABASE_URL" -f data/migrations/009_email_crm_jsonb.sql
psql "$DATABASE_URL" -f data/migrations/010_email_crm_tasks_jsonb.sql
```

Уберите `EMAIL_CRM_PERSISTENCE=memory`, если нужна персистентность в Postgres.

## 3. Ingest (локально)

```bash
curl -sS -X POST "http://localhost:3000/api/cron/email-processor" \
  -H "Authorization: Bearer $CRON_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"action":"poll"}'
```

## 4. SMTP smoke test

4. После успеха: `EMAIL_AUTO_SEND_ENABLED=true`.

## 5. Продакшн cron

```bash
curl -sS -X POST "$BASE_URL/api/cron/email-processor" \
  -H "Authorization: Bearer ${CRON_SECRET}" \
  -d '{"action":"poll"}'
```

## Устранение неполадок

| Симптом | Проверка |
|---------|----------|
| Пустой inbox | Миграции или `memory`? `ANTHROPIC_API_KEY`? |
| 401 на cron | `CRON_SECRET` в Bearer |
| Send fails | `SMTP_*`, порт 587 |
