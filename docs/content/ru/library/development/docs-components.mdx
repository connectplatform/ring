# Компоненты документации

Документация Ring — **MDX-файлы**, рендеримые `next-mdx-remote/rsc` с общей картой компонентов в `components/docs/mdx-docs-shared.tsx`. Используйте эти блоки для страниц, понятных **разработчикам** (пути, код) и **руководителям** (результат, одна диаграмма, ясная навигация).

<Callout type="info">
  **Канонические пути:** контент в `docs/content/{locale}/library/{section}/{page}.mdx`. Публичные URL без `library` — например `/docs/architecture`, `/docs/development/docs-components`.
</Callout>

## Конвейер рендеринга

<Mermaid title="Путь рендера доков">
{`flowchart LR
  MDX["MDX + frontmatter"]
  RSC["next-mdx-remote/rsc"]
  GFM["remark-gfm"]
  Rehype["rehype: Code + Mermaid"]
  Map["docsMdxComponents"]
  Page["/{locale}/docs/..."]

  MDX --> RSC
  GFM --> RSC
  Rehype --> RSC
  RSC --> Map --> Page`}
</Mermaid>

| Этап | Расположение | Роль |
|------|--------------|------|
| Контент | `docs/content/{locale}/library/` | `title`, `description`, тело MDX |
| Resolver | `lib/docs/resolve-doc-file-path.ts` | `/docs/foo` → `library/foo.mdx` |
| Компоненты | `components/docs/mdx-docs-shared.tsx` | Регистрация JSX-тегов |
| Подсветка | Shiki (`nord` / `tokyo-night`) | Серверный `<Code>` |

## Быстрый справочник компонентов

| Компонент | Лучше всего для | Client / server |
|-----------|-----------------|-----------------|
| **Callout** | Резюме для руководителей, предупреждения (`DB_BACKEND_MODE`, legal) | Client |
| **Mermaid** | Одна системная карта + одна sequence на хаб (без разрастания) | Client |
| **Tabs / Tab** | Разработчик vs Оператор (или CEO) на одной странице | Client |
| **Cards / Card** | Навигация хаба к дочерним статьям | Server-friendly |
| **Steps / Step** | Установка и деплой по шагам | Server-friendly |
| **Code** | Конфиг с Shiki | Server (async) |
| **MindMap** | Деревья понятий — осторожно на хабах | Client |
| **Timeline** | История релизов / миграций | Client |
| **RingAISynapseFlow** | Маркетинг / AI-демо — тяжёлый бандл | Client |
| **CodeSandbox** | Живые примеры в `/examples` | Client |
| **Math / MathBlock** | Научный редактор, токеномика | Client |

Огороженные блоки \`\`\`typescript и \`\`\`mermaid преобразуются в `<Code>` и `<Mermaid>` через rehype.

## Callout

Типы: `info` | `warning` | `error` | `success`. Опционально `title`.

<Callout type="success" title="Паттерн executive summary">
  Начинайте хабы с одного абзаца результата для операторов и CEO, затем таблица и одна системная диаграмма.
</Callout>

## Mermaid

На хабах — **одна** архитектурная карта и **одна** sequence. Остальное — на дочерних страницах.

## Timeline, RingAISynapseFlow, CodeSandbox, Math

| Компонент | Когда |
|-----------|-------|
| **Timeline** | Миграции, roadmap (`react-chrono`, client-only) |
| **RingAISynapseFlow** | Только маркетинг — не на справочных страницах |
| **CodeSandbox** | Раздел `/examples`, Sandpack |
| **Math / MathBlock** | Формулы в научных / tokenomics статьях |

## Правила авторинга

| Правило | Почему |
|---------|--------|
| EN каноничен, UK/RU по LOCALE-GAPS | Без расхождений в env/API |
| Frontmatter `title` + `description` | SEO и сайдбар |
| Без `fumadocs-ui` | Только `components/docs/` |
| `npm run dev` после нового MDX | Ловить ошибки Steps/Mermaid |
