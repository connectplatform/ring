# Управление вендорами

Мультивендорный маркетплейс Ring связывает **entities** (`storeActivated: true`) с профилями вендоров, владением товарами и конфигурацией выплат.

> **EN canonical:** [Vendor Management (EN)](/docs/library/features/erp/vendor-management)

## Жизненный цикл

1. **Онбординг** — `/vendor/start` (slug, брендинг, категории).
2. **Товары** — `/vendor/products` (+ модерация главного магазина).
3. **Заказы** — `/vendor/orders` (фильтр по `vendorSettlements`).
4. **Заработок** — `/vendor/earnings` из коллекции **`settlements`**.

## Комиссия платформы

Базовый % зависит от **tier** вендора (`TIER_BENEFITS`). Переопределения — документ `merchant_configs` (admin).

Реферальная комиссия с продажи использует ту же иерархию, что и [refcodes](/docs/library/features/refcodes).

## Навигация

Блок **VENDOR** в сайдбаре: Dashboard, Products, Orders, Stock, Earnings, Settings.
