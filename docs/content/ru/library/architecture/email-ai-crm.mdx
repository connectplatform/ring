## Модули

| Путь | Роль |
|------|------|
| `services/email/email-processor.ts` | Оркестратор: dedup, poll, ingest |
| `services/email/imap/` | `pollBatch()` для cron |
| `services/email/smtp/email-sender.ts` | Исходящие ответы |
| `features/email-crm/repositories/` | JSONB Contact / Task / Draft |
| `features/email-crm/services/` | Thread, message, analytics, send |

## JSONB модель

Строка платформы: `id`, `data`, `created_at`, `updated_at`.

| Коллекция | Id документа |
|-----------|--------------|
| `email_threads` | RFC thread root / Message-ID |
| `email_contacts` | `contact_<sha256(email)>` |
| `email_messages` | RFC Message-ID |
| `email_drafts` | `draft_<uuid>` |
| `email_tasks` | `task_<uuid>` |
| `email_api_usage` | `req_<timestamp>_<random>` |

Миграции: `009_email_crm_jsonb.sql`, `010_email_crm_tasks_jsonb.sql`.

`EMAIL_CRM_PERSISTENCE=memory` — только dev/tests; иначе Postgres через фабрики сервисов.

## Идемпотентность

| Механизм | Поведение |
|----------|-----------|
| DB dedup | `EmailMessageService.exists(messageId)` |
| Poll mutex | Параллельный `pollInboundBatch` → `{ skipped: true }` |
| pollBatch | Ждёт все `handleEmail` перед disconnect |
| Webhook | `uid: 0` — без IMAP mark-seen |

## Cron vs IDLE

| Режим | Когда |
|-------|-------|
| **poll** (рекомендуется) | k8s CronJob, serverless |
| **start** | Dedicated Node + `instrumentation.ts` |
| HTTP start | Debug: `EMAIL_PROCESSOR_ALLOW_HTTP_START=true` |

## Безопасность

- Cron/webhook: `Bearer $CRON_SECRET` или HMAC `X-Email-Webhook-Signature`.
- Admin API: сессия + `isPlatformAdmin`.
- Inbound: 4 слоя anti-injection до LLM.
