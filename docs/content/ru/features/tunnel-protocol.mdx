# 🚇 Протокол Tunnel

**Мультитранспортная Коммуникация Реального Времени Ring**

Протокол Tunnel - это интеллектуальная система коммуникации реального времени Ring Platform - **аналог Ring для Firebase Realtime Database**. Обеспечивает унифицированные сообщения реального времени через WebSocket, Server-Sent Events (SSE), Supabase Realtime и Long-Polling с автоматическим переключением и совместимостью с Edge Runtime.

**v1.6.0:** `lib/tunnel/disconnected-tunnel-context.ts` сохраняет UX офлайн; сочетайте с [FCM push](/docs/library/features/push-notifications-fcm).

## 🎯 Почему Протокол Tunnel?

### Проблема с Firebase RTDB

Firebase Realtime Database предоставляет отличные возможности реального времени, но создаёт зависимость от поставщика:

- ❌ **Привязка к Поставщику**: Зависимость от инфраструктуры Firebase
- ❌ **Несовместимость с Edge Runtime**: Не работает на Vercel Edge
- ❌ **Масштабирование Стоимости**: Ценообразование за операцию
- ❌ **Ограниченный Контроль**: Невозможно оптимизировать под конкретные случаи
- ❌ **Один Транспорт**: Только WebSocket

### Решение Ring: Протокол Tunnel

✅ **Абстракция Транспорта**: Единый API для 5+ транспортов  
✅ **Совместимость с Edge Runtime**: Поддержка SSE для Vercel Edge  
✅ **Ноль Зависимости**: Используйте любой бэкенд (PostgreSQL, Supabase, Firebase)  
✅ **Интеллектуальный Фолбэк**: Автоматическое переключение при сбоях  
✅ **Оптимизация Стоимости**: Самохостинг, платите только за инфраструктуру  
✅ **Поддержка Анонимных**: Функции реального времени для неавторизованных пользователей
