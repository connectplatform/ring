# Мульти-доменне white-label розгортання

Налаштуйте декілька окремих white-label інстансів платформи Ring з різними доменами, брендингом та конфігураціями для різних клієнтів чи ринків.

## 🏢 Підхід до мульти-доменності

Платформа Ring підтримує мульти-доменне розгортання через:
- **Конфігурація за доменом** - Різні налаштування для кожного домену
- **Змінні середовища** - Окремі env файли для кожного інстансу
- **База даних** - Спільна або ізольована залежно від потреб
- **Масштабованість** - Одне розгортання обслуговує декілька брендів

### Крок 3: Middleware для маршрутизації
// middleware.ts

<Code language="typescript" title="TypeScript">
{`import { NextRequest, NextResponse } from 'next/server'
import { domainConfigs } from './config/domains'

export function middleware(request: NextRequest) {
  const hostname = request.headers.get('host') || ''
  const config = domainConfigs[hostname]

  if (config) {
    // Встановити cookies або headers для ідентифікації домену
    const response = NextResponse.next()
    response.cookies.set('domain-config', JSON.stringify(config))
    return response
  }

  // Дефолтний домен
  return NextResponse.next()
}`}
</Code>

### Крок 4: Провайдер конфігурації домену
// providers/DomainProvider.tsx

<Code language="typescript" title="TypeScript">
{`'use client'

import { createContext, useContext, useEffect, useState } from 'react'

interface DomainContextType {
  config: DomainConfig | null
  isLoading: boolean
}

const DomainContext = createContext<DomainContextType>({
  config: null,
  isLoading: true
})

export function DomainProvider({ children }: { children: React.ReactNode }) {
  const [config, setConfig] = useState<DomainConfig | null>(null)
  const [isLoading, setIsLoading] = useState(true)

  useEffect(() => {
    // Отримати конфігурацію з cookies або API
    const getDomainConfig = async () => {
      try {
        const response = await fetch('/api/domain-config')
        const domainConfig = await response.json()
        setConfig(domainConfig)
      } catch (error) {
        console.error('Failed to load domain config:', error)
      } finally {
        setIsLoading(false)
      }
    }

    getDomainConfig()
  }, [])

  return (
    <DomainContext.Provider value={{ config, isLoading }}>
      {children}
    </DomainContext.Provider>
  )
}

export function useDomain() {
  return useContext(DomainContext)
}`}
</Code>

### Динамічне завантаження стилів
// lib/branding.ts

<Code language="typescript" title="TypeScript">
{`export function loadDomainBranding(config: DomainConfig) {
  if (typeof document === 'undefined') return

  const root = document.documentElement

  // Встановити CSS змінні
  root.style.setProperty('--brand-primary', config.colors.primary)
  root.style.setProperty('--brand-secondary', config.colors.secondary)
  root.style.setProperty('--brand-accent', config.colors.accent)

  // Оновити title та meta теги
  document.title = config.brandName
  const metaDescription = document.querySelector('meta[name="description"]')
  if (metaDescription) {
    metaDescription.setAttribute('content', config.brandTagline)
  }
}`}
</Code>

## 🚀 Розгортання мульти-доменності

### Nginx конфігурація
/etc/nginx/sites-available/multi-domain

<Code language="nginx" title="configuration">
{`server {
    listen 80;
    server_name client1.yourplatform.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name client2.yourplatform.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}`}
</Code>

services:

## 💡 Приклади використання

### SaaS платформа
Кожен клієнт отримує власний white-label додаток з повною ізоляцією даних та кастомним брендингом.

### Регіональні ринки
Різні домени для різних країн з локалізованою версіями та регуляторними вимогами.

### Корпоративні клієнти
Великі організації з декількома підрозділами, кожен з власним брендингом та конфігурацією.

---

**🏢 Масштабованість та гнучкість**

Мульти-доменна архітектура дозволяє обслуговувати декілька клієнтів з однієї кодової бази, забезпечуючи повну ізоляцію та кастомізацію для кожного.
