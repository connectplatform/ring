# API повідомлень

Платформа Ring надає комплексну систему повідомлень із **9 REST кінцевими точками** та доставкою через **Tunnel Protocol** (native WSS, SSE, poll). Система підтримує push-сповіщення FCM, налаштування користувачів, масові операції та детальну аналітику.

<Callout type="info" title="🔥 Функції реального часу">
**Tunnel Protocol** — `publishToUserTunnel` на `notifications:unread`, клієнт `useRealtimeNotifications`. k8s: `/api/tunnel/ws`.
</Callout>

## 🏗️ Архітектура системи

### Потік повідомлень

```
Подія тригер → Сервіс повідомлень → Доставка у режимі реального часу → Налаштування користувачів → Push FCM → Аналітика
```

### Стратегія зберігання

- **Первинне**: PostgreSQL з оптимізованими запитами
- **Кешування**: Redis для зберігання сесій (необов'язково)
- **Реальний час**: TunnelHub WSS / SSE / poll через `publishToUserTunnel`
- **Push**: Firebase Cloud Messaging (FCM) інтеграція

## 📋 Довідка кінцевих точок API

### `GET /api/notifications`

Список повідомлень користувачів із розширеним фільтруванням та пагінацією.

#### Параметри

| Параметр | Тип | Обов'язковий | Опис |
|-----------|-----|--------------|------|
| `page` | number | Ні | Номер сторінки (за замовчуванням: 1) |
| `limit` | number | Ні | Елементів на сторінку (за замовчуванням: 20, макс: 100) |
| `status` | string | Ні | Фільтр за статусом: `unread`, `read`, `archived` |
| `type` | string | Ні | Фільтр за типом: `OPPORTUNITY`, `MESSAGE`, `ENTITY`, `SYSTEM`, `WALLET` |
| `priority` | string | Ні | Фільтр за пріоритетом: `low`, `normal`, `high`, `urgent` |
| `before` | string | Ні | ISO дата - показати повідомлення до цієї дати |
| `after` | string | Ні | ISO дата - показати повідомлення після цієї дати |

#### Приклад запиту

<Code language="bash" title="curl -X GET "http://localhost:3000/api/notifications?page=1&limit=10&status=unread&type=OPPORTUNITY" \">
{`-H "Authorization: Bearer your-session-token"`}
</Code>

## 📱 Push-сповіщення

### Інтеграція FCM

Платформа Ring інтегрується з Firebase Cloud Messaging для кросплатформних push-сповіщень.

#### Налаштування клієнта

// Ініціалізація Firebase

<Code language="typescript" title="TypeScript">
{`import { initializeApp } from 'firebase/app'
import { getMessaging, getToken, onMessage } from 'firebase/messaging'

const app = initializeApp(firebaseConfig)
const messaging = getMessaging(app)

// Запит дозволу та отримання токена
const token = await getToken(messaging, {
  vapidKey: 'your-vapid-key'
})

// Реєстрація токена на Ring Platform
await fetch('/api/notifications/fcm/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ token, deviceType: 'web' })
})

// Обробка вхідних повідомлень
onMessage(messaging, (payload) => {
  console.log('Отримано push-сповіщення:', payload)
  showNotification(payload.notification)
})`}
</Code>

#### Service Worker

// public/firebase-messaging-sw.js

<Code language="javascript" title="JavaScript">

## 🔒 Заходи безпеки

### Аутентифікація
- Усі кінцеві точки вимагають дійсної аутентифікації сесії
- Адмін кінцеві точки вимагають ролі `ADMIN`
- FCM токени зашифровані та безпечно збережені

### Обмеження швидкості
- Стандартні користувачі: 100 запитів/хвилину
- Адмін користувачі: 500 запитів/хвилину
- Масові операції: 10 операцій/хвилину

### Валідація даних
- Усі вхідні дані валідуються за допомогою схем Zod
- HTML контент санітізується для запобігання XSS
- Завантаження файлів сканується на віруси та шкідливе ПЗ

### Конфіденційність
- Повідомлення поважають налаштування користувачів
- Ніякої конфіденційної інформації у корисних даних push-сповіщень
- GDPR-сумісна обробка даних

## 🎛️ Конфігурація

### Змінні середовища

Конфігурація FCM Налаштування повідомлень Redis (необов'язково)

<Code language="bash" title="terminal">
{`FCM_SERVER_KEY=your-fcm-server-key
FCM_VAPID_KEY=your-vapid-public-key

NOTIFICATION_RATE_LIMIT=100
NOTIFICATION_BULK_LIMIT=50
NOTIFICATION_RETENTION_DAYS=90

REDIS_URL=redis://localhost:6379`}
</Code>

### Схема бази даних

<Code language="sql" title="-- Таблиця повідомлень">
{`CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id),
  type VARCHAR(50) NOT NULL,
  title VARCHAR(255) NOT NULL,
  message TEXT NOT NULL,
  data JSONB,
  priority VARCHAR(20) DEFAULT 'normal',
  status VARCHAR(20) DEFAULT 'unread',
  created_at TIMESTAMP DEFAULT NOW(),
  read_at TIMESTAMP,
  expires_at TIMESTAMP
);

-- Налаштування користувачів
CREATE TABLE notification_preferences (
  user_id UUID PRIMARY KEY REFERENCES users(id),
  email_notifications BOOLEAN DEFAULT true,
  push_notifications BOOLEAN DEFAULT true,
  preferences JSONB DEFAULT '{}',
  updated_at TIMESTAMP DEFAULT NOW()
);`}
</Code>

---

*Система повідомлень платформи Ring забезпечує надійність корпоративного рівня з доставкою у режимі реального часу, комплексною аналітикою та кросплатформною сумісністю.*
