# 🚇 Протокол Tunnel

**Мультитранспортна Комунікація Реального Часу Ring**

Протокол Tunnel - це інтелектуальна система комунікації реального часу Ring Platform - **аналог Ring для Firebase Realtime Database**. Забезпечує уніфіковані повідомлення реального часу через WebSocket, Server-Sent Events (SSE), Supabase Realtime та Long-Polling з автоматичним перемиканням та сумісністю з Edge Runtime.

**v1.6.0:** `lib/tunnel/disconnected-tunnel-context.ts` зберігає UX офлайн; поєднуйте з [FCM push](/docs/features/push-notifications-fcm).

## 🎯 Чому Протокол Tunnel?

### Проблема з Firebase RTDB

Firebase Realtime Database надає чудові можливості реального часу, але створює залежність від постачальника:

- ❌ **Прив'язка до Постачальника**: Залежність від інфраструктури Firebase
- ❌ **Несумісність з Edge Runtime**: Не працює на Vercel Edge
- ❌ **Масштабування Вартості**: Ціноутворення за операцією
- ❌ **Обмежений Контроль**: Неможливо оптимізувати під конкретні випадки
- ❌ **Один Транспорт**: Тільки WebSocket

### Рішення Ring: Протокол Tunnel

✅ **Абстракція Транспорту**: Єдиний API для 5+ транспортів  
✅ **Сумісність з Edge Runtime**: Підтримка SSE для Vercel Edge  
✅ **Нуль Залежності**: Використовуйте будь-який бекенд (PostgreSQL, Supabase, Firebase)  
✅ **Інтелектуальний Фолбек**: Автоматичне перемикання при збоях  
✅ **Оптимізація Вартості**: Самохостинг, платите тільки за інфраструктуру  
✅ **Підтримка Анонімних**: Функції реального часу для неавторизованих користувачів
