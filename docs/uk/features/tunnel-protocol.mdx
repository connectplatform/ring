# Протокол Tunnel

Realtime-шар Ring Platform: inbox користувача (сповіщення, баланс) і topic-канали (чат, discovery, matcher).

- **Сервер:** `server.ts` — Next.js + native WSS (`/api/tunnel/ws`) при `RING_DEPLOY_TARGET=k8s|self-hosted`
- **Брокер:** `lib/tunnel/hub/` → `InMemoryTunnelHub`
- **Клієнт:** `TunnelProvider` / `useTunnel` → WSS → SSE → poll

## Маршрути API

| Маршрут | Метод | Призначення |
|---------|-------|-------------|
| `/api/tunnel/ws` | WS | Native WebSocket (k8s/self-hosted) |
| `/api/tunnel/sse` | GET | SSE-потік |
| `/api/tunnel/token` | POST | JWT для tunnel auth |
| `/api/tunnel/poll` | GET/POST | Long-polling fallback |

## Конфігурація

```bash
RING_DEPLOY_TARGET=k8s
NEXT_PUBLIC_RING_DEPLOY_TARGET=k8s
# Docker-образ запікає k8s
```
