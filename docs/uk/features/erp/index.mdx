# Ring ERP

Ring ERP відкриває операторам те, що вже працює в бекенді магазину: **склад**, **синхронізацію залишків**, **розрахунки з вендорами** та **sales-assist** для реферальних продажів.

> **EN canonical:** [Ring ERP (EN)](/docs/features/erp/index)

## Фази дорожньої карти

| Фаза | Статус | Зміст |
|------|--------|-------|
| 1 | **Поточна** | Admin `/admin/store` (Замовлення · Склад · Комісії), меню вендора |
| 2 | Активна | Per-product referral %, спільний resolver з refcodes |
| 3 | Заплановано | FSMA traceability UI, AI dashboards |

## Сервіси

| Сервіс | Шлях | Роль |
|--------|------|------|
| Склад | `erp-stock-service.ts` | Списання при оплаті, рухи, алерти |
| Синхронізація | `inventory-sync.ts` | Резерви та трансфери |
| Розрахунки | `settlement.ts` + `settlement-pipeline.ts` | Ledger `settlements` |
| Дії ERP | `app/_actions/store-erp.ts` | Sales-assist, якість |

## Поверхні UI

- **Адмін:** `/admin/store` — вкладки замовлень, складу, комісій.
- **Вендор:** `/vendor/dashboard` → замовлення, склад, заробіток, налаштування.
