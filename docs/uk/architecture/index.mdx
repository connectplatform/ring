# Архітектура

<Callout type="info">
  **Для керівників:** Ring — **white-label платформа**: один код, багато брендованих деплоїв. Продакшен-кільця працюють на **PostgreSQL-primary** у Kubernetes, Firebase лише для **push**, платежі через **PaymentConductor**, локалізація без циклів редіректів авторизації.
</Callout>

Ring Platform v1.6.0 — **багатотенантний Next.js застосунок** з єдиним контрактом `DatabaseService`, конфігурованими платежами та опційним Web3/токенним шаром на клон. Ця сторінка — **хаб архітектури**; кожна тема має окрему глибоку статтю.

## Платформа одним поглядом

| Вимір | Підхід Ring |
|-------|-------------|
| **Застосунок** | Next.js 16 App Router, React 19 Server Components, Tailwind 4 |
| **Дані (продакшен)** | `DB_BACKEND_MODE=k8s-postgres-fcm` — Postgres + PostGIS, JSONB-first |
| **Автентифікація** | Auth.js v5 — `auth()` на рівні layout, тонкий `proxy.ts` лише для локалі |
| **Платежі** | PaymentConductor — WayForPay, Stripe, ledger + webhooks |
| **Realtime** | Tunnel protocol — WebSocket / SSE / опційний канал Supabase |
| **Файли** | Vercel Blob або сховище клону |
| **Деплой** | k3s (кільця ringdom.org) — не Vercel Postgres |
| **Розширення** | White-label env, portal config, токен-контракти на клон |

**Орієнтири в коді**

| Тема | Де читати |
|------|-----------|
| Locale proxy | `proxy.ts`, `lib/proxy-intl.ts` |
| Auth enforcement | `(authenticated)/[locale]/layout.tsx`, `auth.config.ts` |
| Режим БД | `lib/database/backend-mode-config.ts` |
| Платежі | `features/store/services/payment-conductor*` |
| Схема SSOT | `data/schema.sql` |

**Шари компонентів**

- **Hooks** — дані, кредит гаманця, realtime підписки
- **Feature components** — store, messages, wallet (повторне використання)
- **Pages** — композиція features; без прямих викликів адаптерів у UI

</Tab>

<Tab value="Оператори">

### Чеклист деплою

1. **`DB_BACKEND_MODE`** — `k8s-postgres-fcm` для self-hosted Postgres (канонічне ім’я enum).
2. **Postgres** + міграції (`getting-started/migrations`).
3. **FCM** — якщо потрібен push; Firebase **не** основна БД у Postgres-режимі.
4. **PaymentConductor** — WayForPay / Stripe, webhook URLs.
5. **Ingress** на Next.js; секрети в k8s.
6. Опційно: контракти **RING** / **ReferralRewards**; `NEXT_PUBLIC_RING_TOKEN_ADDRESS`.

### Чого ми *не* використовуємо

- `@vercel/postgres` як основний SQL драйвер
- Auth у `proxy.ts` (цикли редіректів — zero-loop proxy)
- Firestore як source of truth при `k8s-postgres-fcm`

### Сигнали моніторингу

- API p95, невдалі payment webhooks, помилки продовження підписок
- Насичення пулу з’єднань БД, drift версій міграцій
- Помилки FCM токенів, відключення Tunnel

## Сквозні потоки

### Комерція + членство

Checkout через **PaymentConductor** (fiat) або **RING credit / on-chain** підписку. Settlement і комісії — у [PaymentConductor](/docs/architecture/payment-conductor) та store guides.

### Контент + локаль

Публічні сторінки — **next-intl**. Документація та новини — файли `docs/{locale}/`. EN канонічний, якщо LOCALE-GAPS не позначає стислий UK/RU summary.

### White-label клони

Форк репо, portal branding, режим БД, опційна токеноміка — [White-label quick start](/docs/customization/quick-start), [Token economics](/docs/customization/token-economics).

<Callout type="success">
  **Базис v1.6.0:** Postgres-primary + PaymentConductor + slim proxy + locale system. Почніть з [Режимів бекенду](/docs/architecture/backend-modes-and-databases), потім [Environment](/docs/deployment/environment).
</Callout>

<Callout type="warning">
  **`DB_HYBRID_MODE` ≠ `DB_BACKEND_MODE`.** Для нових деплоїв використовуйте `DB_BACKEND_MODE`; legacy hybrid — на сторінці backend-modes.
</Callout>
