# 🔐 Архитектура Аутентификации

**Система Аутентификации Ring Platform** - мультипровайдерная аутентификация на базе Auth.js v5 с расширенной безопасностью, ролевым контролем доступа и бесшовной интеграцией Web3.

---

## 📋 Обзор

Ring Platform реализует **сложную многоуровневую архитектуру аутентификации**, объединяющую традиционные OAuth провайдеры, magic links без паролей и передовую аутентификацию через Web3 кошельки с **революционной PIN-безопасностью** (без seed-фраз).

### Ключевые Особенности

- ✅ **5 Провайдеров Аутентификации** - Google (GIS + OAuth), Apple, Magic Links, Крипто-кошельки, PIN-безопасность
- ✅ **Современный Auth.js v5** - Новейший фреймворк аутентификации с совместимостью edge runtime
- ✅ **5-Уровневая Иерархия Ролей** - VISITOR → SUBSCRIBER → MEMBER → CONFIDENTIAL → ADMIN
- ✅ **Поддержка Многих Бэкендов** - Работает с k8s-postgres-fcm, firebase-full и supabase-fcm режимами
- ✅ **Web3 Без Сложности** - Пользователи авторизуются социально, получают Web3 кошельки автоматически
- ✅ **Система PIN-Безопасности** - Революционный Web3 доступ без seed-фраз
- ✅ **GDPR/CCPA Совместимость** - 30-дневный период восстановления при удалении аккаунта
- ✅ **Связывание Email** - Автоматическое связывание аккаунтов с одинаковым email
- ✅ **KYC Интеграция** - Загрузка документов через Vercel Blob storage

## 🌐 Провайдеры Аутентификации

### 1. Google Аутентификация (Двойной Режим)

**Традиционный OAuth + Google Identity Services (GIS)**

<Mermaid>
{`sequenceDiagram
    participant User as Пользователь
    participant Client as Next.js Client
    participant AuthJS as Auth.js v5
    participant Google as Google OAuth
    participant DB as PostgreSQL/Firebase
    
    Note over User,DB: Традиционный OAuth Поток
    User->>Client: Клик "Войти через Google"
    Client->>AuthJS: Инициировать OAuth
    AuthJS->>Google: Редирект на согласие
    Google->>User: Показать экран согласия
    User->>Google: Авторизовать
    Google->>AuthJS: Вернуть auth code
    AuthJS->>Google: Обменять code на token
    Google-->>AuthJS: Access token + данные пользователя
    AuthJS->>DB: Сохранить/обновить пользователя
    AuthJS->>Client: Установить session cookie
    Client->>User: Редирект на профиль
    
    Note over User,DB: GIS One Tap Поток (Быстрее)
    User->>Client: Страница загружается
    Client->>Client: Загрузить GIS SDK
    Client->>User: Показать One Tap промпт
    User->>Client: Нажать для входа
    Client->>AuthJS: Отправить JWT credential
    AuthJS->>Google: Верифицировать JWT token
    Google-->>AuthJS: Token валидный + данные пользователя
    AuthJS->>DB: Сохранить/обновить пользователя
    AuthJS->>Client: Установить session cookie
    Client->>User: Мгновенная аутентификация`}
</Mermaid>

**Реализация:**
- **Традиционный OAuth:** Полный редирект поток для максимальной совместимости
- **GIS One Tap:** Клиентский popup для мгновенной аутентификации
- **Поддержка Темы:** GIS кнопка динамически меняет outline (светлая) / filled_black (темная)
- **Связывание Email:** Аккаунты с одинаковым email автоматически связываются

### 3. Magic Links (Без Паролей)

**Email-Базированная Безопасная Аутентификация**

<Mermaid>
{`sequenceDiagram
    participant User as Пользователь
    participant Client as Next.js App
    participant AuthJS as Auth.js v5
    participant Resend as Resend API
    participant DB as База Данных
    
    User->>Client: Ввести email адрес
    Client->>AuthJS: Запрос magic link
    AuthJS->>DB: Сгенерировать токен верификации
    DB-->>AuthJS: Токен сохранен
    AuthJS->>Resend: Отправить email со ссылкой
    Resend->>User: Получить email
    User->>Resend: Клик на magic link
    Resend->>Client: Редирект на callback
    Client->>AuthJS: Верифицировать токен
    AuthJS->>DB: Валидировать токен
    DB-->>AuthJS: Токен валидный
    AuthJS->>DB: Использовать токен (одноразовый)
    AuthJS->>Client: Установить session cookie
    Client->>User: Аутентифицировано`}
</Mermaid>

**Функции Безопасности:**
- **Ограничены По Времени:** Токены истекают через 15 минут
- **Одноразовое Использование:** Автоматическая инвалидация после успешного использования
- **Верификация Email:** Обязательна для создания аккаунта
- **GDPR Совместимость:** Без хранения паролей

### Определение Ролей

| Роль | Уровень | Доступ | Случаи Использования |
|------|---------|--------|---------------------|
| **VISITOR** | 0 | Публичный контент, просмотр организаций/возможностей | Неаутентифицированные пользователи |
| **SUBSCRIBER** | 1 | Создание возможностей, базовые сообщения | Бесплатные зарегистрированные пользователи |
| **MEMBER** | 2 | Создание организаций, vendor функции, NFT маркетплейс | Платный тариф ($29/мес), бизнесы |
| **CONFIDENTIAL** | 3 | Доступ к конфиденциальным организациям/возможностям | Верифицированные организации |
| **ADMIN** | 4 | Полный системный доступ, управление пользователями | Администраторы платформы |
