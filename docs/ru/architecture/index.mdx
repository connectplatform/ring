# Архитектура

<Callout type="info">
  **Для руководителей:** Ring — **white-label платформа**: один код, много брендированных деплоев. Продакшен-кольца работают на **PostgreSQL-primary** в Kubernetes, Firebase только для **push**, платежи через **PaymentConductor**, локализация без циклов редиректов авторизации.
</Callout>

Ring Platform v1.6.0 — **мультитенантное Next.js приложение** с единым контрактом `DatabaseService`, конфигурируемыми платежами и опциональным Web3/токенным слоем на клон. Эта страница — **хаб архитектуры**; каждая тема имеет отдельную глубокую статью.

## Платформа одним взглядом

| Измерение | Подход Ring |
|-----------|-------------|
| **Приложение** | Next.js 16 App Router, React 19 Server Components, Tailwind 4 |
| **Данные (продакшен)** | `DB_BACKEND_MODE=k8s-postgres-fcm` — Postgres + PostGIS, JSONB-first |
| **Аутентификация** | Auth.js v5 — `auth()` на уровне layout, тонкий `proxy.ts` только для локали |
| **Платежи** | PaymentConductor — WayForPay, Stripe, ledger + webhooks |
| **Realtime** | TunnelHub — native WSS / SSE / poll; `RING_DEPLOY_TARGET` |
| **Файлы** | Vercel Blob или хранилище клона |
| **Деплой** | k3s (кольца ringdom.org) — не Vercel Postgres |
| **Расширение** | White-label env, portal config, токен-контракты на клон |

**Ориентиры в коде**

| Тема | Где читать |
|------|------------|
| Locale proxy | `proxy.ts`, `lib/proxy-intl.ts` |
| Auth enforcement | `(authenticated)/[locale]/layout.tsx`, `auth.config.ts` |
| Режим БД | `lib/database/backend-mode-config.ts` |
| Платежи | `features/store/services/payment-conductor*` |
| Схема SSOT | `data/schema.sql` |

**Слои компонентов**

- **Hooks** — данные, кредит кошелька, realtime подписки
- **Feature components** — store, messages, wallet (повторное использование)
- **Pages** — композиция features; без прямых вызовов адаптеров в UI

<Tab value="Операторы">

### Чеклист деплоя

1. **`DB_BACKEND_MODE`** — `k8s-postgres-fcm` для self-hosted Postgres (каноническое имя enum).
2. **Postgres** + миграции (`getting-started/migrations`).
3. **FCM** — если нужен push; Firebase **не** основная БД в Postgres-режиме.
4. **PaymentConductor** — WayForPay / Stripe, webhook URLs.
5. **Ingress** на Next.js; секреты в k8s.
6. Опционально: контракты **RING** / **ReferralRewards**; `NEXT_PUBLIC_RING_TOKEN_ADDRESS`.

### Чего мы *не* используем

- `@vercel/postgres` как основной SQL драйвер
- Auth в `proxy.ts` (циклы редиректов — zero-loop proxy)
- Firestore как source of truth при `k8s-postgres-fcm`

### Сигналы мониторинга

- API p95, неудачные payment webhooks, ошибки продления подписок
- Насыщение пула соединений БД, drift версий миграций
- Ошибки FCM токенов, отключения Tunnel

## Сквозные потоки

### Коммерция + членство

Checkout через **PaymentConductor** (fiat) или **RING credit / on-chain** подписку. Settlement и комиссии — в [PaymentConductor](/docs/architecture/payment-conductor) и store guides.

### Контент + локаль

Публичные страницы — **next-intl**. Документация и новости — файлы `docs/{locale}/`. EN каноничен, если LOCALE-GAPS не отмечает краткий UK/RU summary.

### White-label клоны

Форк репо, portal branding, режим БД, опциональная токеномика — [White-label quick start](/docs/customization/quick-start), [Token economics](/docs/customization/token-economics).

<Callout type="success">
  **Базис v1.6.0:** Postgres-primary + PaymentConductor + slim proxy + locale system. Начните с [Режимов бэкенда](/docs/architecture/backend-modes-and-databases), затем [Environment](/docs/deployment/environment).
</Callout>

<Callout type="warning">
  **`DB_HYBRID_MODE` ≠ `DB_BACKEND_MODE`.** Для новых деплоев используйте `DB_BACKEND_MODE`; legacy hybrid — на странице backend-modes.
</Callout>
