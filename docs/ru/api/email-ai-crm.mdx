# Email AI-CRM API

> **EN canonical:** полные тела запросов, примеры curl и JSON webhook — [Email AI-CRM API (EN)](/docs/api/email-ai-crm)

<Callout type="warning" title="Доступ админа">
Все `/api/admin/email/*` требуют сессии **platform admin** (`admin` или `superadmin`).
</Callout>

<Callout type="info" title="Cron / webhook">
`/api/cron/*` и `/api/webhooks/email/inbound` — `CRON_SECRET` или `WEBHOOK_EMAIL_SECRET`, не user session.
</Callout>

## Cron — email processor

### `POST` или `GET` `/api/cron/email-processor`

**Auth:** `Authorization: Bearer $CRON_SECRET`

| `action` | Поведение |
|----------|-----------|
| `poll` (default) | `pollInboundBatch()` — UNSEEN → process → disconnect |
| `status` | Статистика процессора + IMAP |
| `stop` | Остановить IDLE |
| `start` | IDLE (нужен `EMAIL_PROCESSOR_ALLOW_HTTP_START=true`) |
| `mark-overdue-tasks` | Просроченные задачи |

```bash
curl -X POST "$BASE_URL/api/cron/email-processor" \
  -H "Authorization: Bearer $CRON_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"action":"poll"}'
```

### `GET /api/cron/email-analytics`

7-дневный снимок дашборда (как admin analytics).

## Webhook — входящая почта

### `POST /api/webhooks/email/inbound`

**Auth:** `Bearer $WEBHOOK_EMAIL_SECRET` или HMAC-SHA256 в `X-Email-Webhook-Signature`.

Минимальные поля: `messageId`, `from`, `subject`, `bodyText`. Вызывает `EmailProcessor.ingestEvent()` с `uid: 0`.
