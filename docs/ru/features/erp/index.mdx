# Ring ERP

Ring ERP открывает операторам то, что уже работает в бэкенде магазина: **склад**, **синхронизацию остатков**, **расчёты с вендорами** и **sales-assist** для реферальных продаж.

> **EN canonical:** [Ring ERP (EN)](/docs/features/erp/index)

## Фазы дорожной карты

| Фаза | Статус | Содержание |
|------|--------|------------|
| 1 | **Текущая** | Admin `/admin/store` (Заказы · Склад · Комиссии), меню вендора |
| 2 | Активна | Per-product referral %, общий resolver с refcodes |
| 3 | Запланировано | FSMA traceability UI, AI dashboards |

## Сервисы

| Сервис | Путь | Роль |
|--------|------|------|
| Склад | `erp-stock-service.ts` | Списание при оплате, движения, алерты |
| Синхронизация | `inventory-sync.ts` | Резервы и трансферы |
| Расчёты | `settlement.ts` + `settlement-pipeline.ts` | Ledger `settlements` |
| Действия ERP | `app/_actions/store-erp.ts` | Sales-assist, качество |

## Поверхности UI

- **Админ:** `/admin/store` — вкладки заказов, склада, комиссий.
- **Вендор:** `/vendor/dashboard` → заказы, склад, заработок, настройки.
