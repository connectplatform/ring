# Email AI-CRM

**AI-управляемый CRM для публичных почтовых ящиков** (например `info@example.com`). Входящая почта классифицируется и собирается в треды; Claude формирует черновики ответов; админы утверждают отправку и видят расходы API.

> **EN canonical:** полные диаграммы, таблицы API и runbook — [Email AI-CRM (EN)](/docs/features/email-ai-crm) · [Архитектура](/docs/architecture/email-ai-crm) · [API](/docs/api/email-ai-crm) · [Гайд разработчика](/docs/examples/email-ai-crm)

## Обзор

| Слой | Ответственность |
|------|-----------------|
| **Сбор** | Cron `POST /api/cron/email-processor` (`action: poll`) или webhook |
| **Процессор** | `services/email/email-processor.ts` — parse, security, AI, CRM |
| **Хранение** | PostgreSQL JSONB: `email_threads`, `email_contacts`, `email_messages`, `email_drafts`, `email_tasks`, `email_api_usage` |
| **Исходящие** | `EmailSenderService` (nodemailer), `In-Reply-To` / `References` |
| **Админ UI** | inbox, drafts, contacts, analytics, tasks |

## Бизнес-правила

- **Дедуп Message-ID** — один RFC `Message-ID` не обрабатывается дважды.
- **Auto-send выключен** — `EMAIL_AUTO_SEND_ENABLED=false` до SMTP smoke test.
- **Cron poll в продакшне** — каждые 1–5 мин; IDLE только для долгоживущего Node (`EMAIL_PROCESSOR_AUTOSTART`).
- **Admin API** — `auth()` + `isPlatformAdmin` на `/api/admin/email/*`.

## Локальная конфигурация

Добавьте в **`.env.local`** (см. `env.local.template`, блок **EMAIL CRM**):

```bash
# .env.local
IMAP_HOST=mail.example.com
IMAP_USER=info@example.com
IMAP_PASSWORD=

SMTP_HOST=mail.example.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=info@example.com
SMTP_PASSWORD=

ANTHROPIC_API_KEY=

CRON_SECRET=generate-a-long-random-string
EMAIL_AUTO_SEND_ENABLED=false
EMAIL_CRM_PERSISTENCE=memory
```

| Переменная | Описание |
|------------|----------|
| `IMAP_*` / `SMTP_*` | Ваш community inbox (любой IMAP/SMTP) |
| `ANTHROPIC_API_KEY` | Intent, sentiment, генерация черновиков |
| `CRON_SECRET` | Bearer для `/api/cron/email-processor` |
| `EMAIL_AUTO_SEND_ENABLED` | `false` до SMTP smoke test |
| `EMAIL_CRM_PERSISTENCE` | `memory` для быстрого dev; omit + миграции `009`/`010` для Postgres |

## Продакшн

Те же переменные — в secret store (k8s/Vercel). Миграции `009`+`010`, cron `poll` каждые 1–5 мин. См. [гайд разработчика](/docs/examples/email-ai-crm).
